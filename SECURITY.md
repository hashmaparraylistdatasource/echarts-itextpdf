# Security Policy

## Reporting a vulnerability

If you believe you have found a security vulnerability in `echarts-itextpdf`, please do not
open a public issue with exploit details.

Until a dedicated security contact is published for the public repository, use a private
maintainer contact channel or repository security advisory workflow when available.

When reporting a vulnerability, please include:

- the affected version or commit
- the attack surface or entry point
- steps to reproduce
- impact assessment
- any suggested remediation

We will aim to acknowledge reports promptly and work toward a coordinated fix and disclosure.

## Supported versions

Before the first public release, security support should be assumed for the latest `main`
branch state only.

After versioned releases begin, this policy should be updated to list supported release
lines explicitly.
